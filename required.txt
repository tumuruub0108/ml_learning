AI, ялангуяа машин сургалт (machine learning) болон computer vision салбарт “label” гэдэг нь өгөгдөлд тогтоосон ангилал буюу шошго гэсэн утгатай юм. Өөрөөр хэлбэл, AI системд өгч байгаа өгөгдөлд “энэ юу вэ?” гэдгийг зааж өгдөг тэмдэглэгээ юм.

Товчхондоо: Label гэдэг нь AI-д зааж өгөх “зөв хариу” буюу өгөгдлийг ангилах тэмдэглэгээ юм. AI систем энэ label-үүдийг ашиглаад хэрхэн зөв таамаглахыг сурдаг.


Supervised learning гэж юу вэ

Supervised learning гэдэг нь машин сургалтын нэг төрлийн арга бөгөөд мэдээллийн өгөгдөл (data) болон түүний зөв хариу (label) хоёрыг ашиглан модель сургах арга юм. Энгийнээр хэлэхэд, компьютер тухайн өгөгдөлд ямар үр дүн гарахыг урьдчилан мэддэг үед суралцдаг гэсэн үг.


1. Classification (Ангилах)

Юу вэ: Өгөгдлийг урьдчилан тодорхойлсон ангилалд (category) хуваах процесс.
Гол зорилго: Өгөгдлийг ямар нэг шалгуур ангилал руу хуваах.

Жишээ:

И-мэйл нь spam уу эсвэл not spam уу гэдгийг тодорхойлох.
Зүрхний өвчтэй эсэхийг yes/no ангилалд оруулах.
Цэцгийн төрлийг Setosa, Versicolor, Virginica гэж ангилах.

Төрөл:
Binary classification (2 ангилалтай)
Multi-class classification (олон ангилалтай)



2. Regression (Бууралт)
Юу вэ: Өгөгдлөөс тоо утга (continuous value) таамаглах процесс.
Гол зорилго: Өгөгдлийг тоон утгаар урьдчилан таамаглах.

Жишээ:
Байшингийн үнэ тооцоолох.
Хүний нас, өндөр, жингээс биеийн индекс (BMI) тооцоолох.
Цаг агаарын температурыг урьдчилан таамаглах.


Хамгийн гол ялгаа

| Онцлог          | Classification     | Regression        |
| --------------- | ------------------ | ----------------- |
| Гарах үр дүн    | Ангилал (category) | Тогтмол тоон утга |
| Жишээ           | Spam / Not spam    | Байшингийн үнэ    |
| Өгөгдлийн төрөл | Categorical        | Continuous        |


NOTE
The words target and label are generally treated as synonyms in supervised learning, but
target is more common in regression tasks and label is more common in classification
tasks. Moreover, features are sometimes called predictors or attributes. These terms may
refer to individual samples (e.g., “this car’s mileage feature is equal to 15,000”) or to all
samples (e.g., “the mileage feature is strongly correlated with price”



Unsupervised learning гэж юу вэ

Unsupervised learning гэдэг нь машин сургалтын арга бөгөөд өгөгдөлд зөв хариу (label) байхгүй үед компьютерыг суралцуулдаг арга юм. Энгийнээр хэлэхэд, өгөгдөлд ямар дүрэм, бүтэц байгааг урьдчилан заагаагүй үед машин өөрөө олж мэддэг гэсэн үг.

Онцлог:

Label байхгүй: Өгөгдөлд зөв хариу байхгүй.
Жишээ: Клиентийн худалдан авалтын өгөгдөл
Модель бүтцээ олдог: Өгөгдөл доторхи холбоо, хуваагдал, давтагдал зэргийг автоматаар таньж авдаг.
Зорилго: Өгөгдлийг ангилах, бүлэглэх, багцлах, бүтэцлэх

Жишээ:

Clustering (Бүлэглэх):
Өгөгдөл: Клиентуудын худалдан авалтын зуршил
Зорилго: Адил төстэй хэрэглэгчдийг бүлэглэх
Dimensionality Reduction (Өгөгдлийн хэмжээг багасгах):
Өгөгдөл: Зураг, текст, sensor өгөгдөл
Зорилго: Өгөгдлийг багасгаж, гол мэдээллийг нь авах (жишээ: PCA)




ерөнхий ялгаа
| Онцлог            | Supervised Learning         | Unsupervised Learning                |
| ----------------- | --------------------------- | ------------------------------------ |
| Label (зөв хариу) | Байдаг                      | Байхгүй                              |
| Зорилго           | Урьдчилсан хариуг таамаглах | Өгөгдлийн бүтэц, хуваагдал олж мэдэх |
| Жишээ             | Regression, Classification  | Clustering, Dimensionality Reduction |




Semi-supervised learning гэж юу вэ
Semi-supervised learning гэдэг нь supervised learning ба unsupervised learning хоёрын дунд орших машин сурах арга юм. Үүний гол онцлог нь:
unlabeled
Өгөгдлийн бүтэц:

Зарим өгөгдөл нь labeled (шошготой), бусад нь unlabeled (unlabeled) байдаг.
Жишээ: Та 1000 зурагтай ба тэдгээрийн 100 нь "нохой", "муур" гэх мэт ангилалттай, харин үлдсэн 900 зураг нь unlabeled байж болно.

Зорилго:

Машин суралцахдаа labeled өгөгдлөөс суралцаж, unlabeled өгөгдлийн мэдээллийг ашиглан сургалтын үр дүнг сайжруулах.
Ингэснээр unlabeled өгөгдөл ихтэй үед ч өндөр нарийвчлалтай загвар бий болгох боломжтой.

Арга техник:

Self-training: Загвар unlabeled өгөгдөлд өөрийн таамаглалыг үүсгэн, дараа нь үүнийг ашиглан дахин сургах.
Co-training: Хоёр загвар өөр өөр үзэмжээр өгөгдлийг labeled болгож суралцах.
Graph-based methods: Өгөгдлийг граф хэлбэрт оруулж, төстэй өгөгдлийг бүлэглэн labeled өгөгдлөөс суралцах.

Давуу тал:

labeled өгөгдөл цөөхөн үед ч үр ашигтай.
Өгөгдлийн labeled байх зардлыг бууруулдаг.

Жишээ хэрэглээ:

Вэб хуудсуудын спам имэйл илрүүлэх (ихэнх имэйл unlabeled).
Эмнэлгийн зураглалд өвчний оношлох (ихэнх зураг unlabeled).
Хэл боловсруулалт, текст ангилах.


Self-supervised learning гэж юу вэ
Self-supervised learning (SSL) гэдэг нь машин сургалтын нэг арга бөгөөд label буюу unlabeled өгөгдлийг ашиглан модель өөрөө сургалтанд суралцдаг аргыг хэлдэг. Өөрөөр хэлбэл, өгөгдлийн доторх өөрийн гэсэн мэдээллийг ашиглаж “зөв хариу” буюу сургах дохиог үүсгэдэг.

Энгийнээр тайлбарлавал:

Өгөгдөлд label хийх шаардлагагүй:

Жишээ нь supervised learning-д зураг дээр “нохой” эсвэл “муур” гэсэн label зааж өгдөг.
Self-supervised learning-д зурагны өөр хэсгийг далд болгож, модель нь тухайн хэсгийг таахыг сураад төлөвлөсөн representation-г бий болгодог.
Өөрийгөө “сургадаг” дохио ашиглана:
Өгөгдлийг өөрөө хувилж, хэсэг нь “input”, хэсэг нь “target” болдог.

Жишээ:

Нийтлэг текст: үгсийн заримыг нууж, үлдсэн текстээс тухайн үгийг таах (BERT моделиин masked language modeling).
Зураг: зурагны хэсэгийг нууж, нуугдсан хэсгийг сэргээх (Masked Image Modeling).
Дуу/Аудио: дууны хэсгийг нууж, үлдсэнээс нь таах.

Давуу тал:

labeled өгөгдөл хэрэгтэй болдог supervised learning-тэй харьцуулахад олон тооны unlabeled өгөгдөл ашиглаж чадна.
Өгөгдлийн representation буюу онцлогийг сайн суралцах боломжтой.

Жишээний хэрэглээ:

NLP: BERT, GPT (pretraining фаз)
Computer Vision: SimCLR, MAE
Аудио: Wav2Vec





Reinforcement learning гэж юу вэ
Reinforcement Learning (RL) гэдэг нь машин сургалтын нэг төрөл бөгөөд модель буюу агент орчны (environment) дунд шийдвэр гаргаж, шагнал (reward) авах замаар суралцдаг аргыг хэлдэг.

1. Гол ойлголтууд

Agent (агент): Суралцаж байгаа компьютер програм буюу робот.
Environment (орчин): Агент ажиллаж байгаа орчин буюу тоглоом, роботын орчин, бизнесийн систем гэх мэт.
State (байдал): Агентын одоогийн орчны мэдээлэл.
Action (үйлдэл): Агент хийх шийдвэр.
Reward (шагнал): Агентын хийсэн үйлдэл зөв эсвэл буруу болохыг заадаг тооцоолсон утга.

2. Ажиллах зарчим

Агент орчны одоогийн байдал (state)-ийг харна.
Байдалд тохирсон үйлдэл (action) хийдэг.
Орчин тухайн үйлдэлд шагнал (reward) өгдөг.
Агент шагналын үндсэн дээр цаашид илүү сайн шийдвэр гаргах стратеги (policy) суралцдаг.
Энэ үйлдлийг үзэгдэл → үйлдэл → шагнал → шинэ байдал гэсэн циклд үргэлжлүүлнэ.

3. Онцлог

Supervised learning-ээс ялгаатай:
Supervised learning-д шууд зөв хариу (label) өгөгддөг.
RL-д зөв хариу шууд өгөгдөхгүй, зөвхөн шагналын сигнал байдаг.
Цаашид илүү сайн сурах нь агентын үйлдэл ба шагналын давтамж дээр суурилна.

4. Жишээ хэрэглээ

Тоглоом: AlphaGo, Chess AI
Робот: алхах, зүйл барих
Санхүү: хөрөнгийн зах зээлд арилжааны стратеги сурах
Машин сургалтын simulation: автоном машин жолоодох




Batch learning гэж юу вэ
Batch learning гэдэг нь машины сургалтын (machine learning) нэг арга бөгөөд энэ арга нь бүх сургалтын өгөгдлийг нэг дор ашиглан загварыг сургах үйл явцыг хэлдэг.

Үндсэн ойлголт:

Batch learning-д бид бүх өгөгдлийг нэг удаад (batch буюу багц) загварт оруулж, алдааг (error) тооцож, параметрүүдийг шинэчилдэг.
Өөрөөр хэлбэл, модель бүх өгөгдлийг мэддэг, дараа нь нэг удаа сургалт хийж дуусдаг.

Онцлог шинж:

Загварын шинэчлэлт (update) нь бүх өгөгдлийг ашигласны дараа хийгддэг.
Их хэмжээний өгөгдөлд илүү удаан боловч тогтвортой үр дүн гардаг.
Онлайн буюу incremental learning-тэй харьцуулахад шинэ өгөгдөл орж ирсэн ч даруй сурдаггүй, сургалтыг дахин хийх шаардлагатай.

Жишээ:

Хэрвээ та 10,000 өгөгдлийн бичлэгтэй dataset-тэй бол batch learning-д бүгдийг нэг дор загварт оруулж сургалт хийнэ.

Давуу тал:

Илүү стабил (stable) сургалт.
Gradient descent зэрэг алгоритмд илүү нарийн тохируулга хийх боломжтой.

Сул тал:

Маш их өгөгдөлтэй бол мэдээллийн санах ой (memory) их хэрэг болно.
Шинэ өгөгдөл орж ирэхэд загварыг дахиад сургалт хийх хэрэгтэй.

WARNING
Even a model trained to classify pictures of cats and dogs may need to be retrained
regularly, not because cats and dogs will mutate overnight, but because cameras keep
changing, along with image formats, sharpness, brightness, and size ratios. Moreover,
people may love different breeds next year, or they may decide to dress their pets with tiny
hats—who knows?



Online learning гэж юу вэ
“Online learning” гэдэг нь машин суралцалтын хүрээнд хэрэглэгддэг ойлголт бөгөөд модел нь өгөгдлийг нэг удаа бүхэлд нь авч суралцахын оронд өгөгдлийг хэсэг хэсгээр буюу дараалалтайгаар авч, алхам алхамаар шинэчлэх аргыг хэлнэ.

Ингэхдээ гол онцлог нь:

Өгөгдөл дараалалтай орж ирдэг
Модел нь бүх dataset-ийг нэг дор харахгүй, харин өгөгдөл ирэхийн хэрээр шинэ мэдлэгийг сурдаг.

Модел байнга шинэчлэгддэг
Өмнөх сурсан мэдлэг дээрээ суурилж, шинэ мэдээллийг нэмж суралцдаг тул реал-тайм буюу шууд шинэчлэгддэг.

Давуу тал

Их хэмжээний өгөгдөлтэй ажиллахад тохиромжтой (all-at-once сургалт хэтэрхий их ресурс шаарддаг).
Өгөгдөл цаг хугацааны явцад өөрчлөгдөж байвал шинэчилсэн мэдлэг хурдан авдаг.

Сул тал

Өмнөх өгөгдлийг мартчихдаг (model нь бүх dataset-ийг хадгалдаггүй бол).
Дарааллын дарааллаас шалтгаалж сурсан үр дүн нь тогтворгүй байж болно.


Жишээ:

Банкны зээлийн систем шинэ хэрэглэгчдэд зээл олгох эрсдэлийг үнэлэхдээ шинэ хэрэглэгч бүрийн өгөгдлийг орж ирэхийн хэрээр моделээ шинэчилж сурдаг.

Рекомендацийн систем (Netflix, YouTube) шинэ хэрэглэгчийн үзсэн контентыг үндэслэн шууд зөвлөмжийг шинэчилд


Online learning системийн нэг чухал үзүүлэлт бол тэдгээр нь өөрчлөгдөж буй өгөгдөлд хэр хурдан дасан зохицож байх ёстой вэ гэвэл үүнийг сургалтын хурд(learning rate.) гэж нэрлэдэг.

WARNING
Out-of-core learning is usually done offline (i.e., not on the live system), so online
learning can be a confusing name. Think of it as incremental learning

A big challenge with online learning is that if bad data is fed to the system,
the system’s performance will decline, possibly quickly (depending on thedata quality and learning rate).

Instance-Based Versus Model-Based Learning
One more way to categorize machine learning systems is by how they generalize. Most machine learning tasks are about making predictions. This means that given a number of training examples, the system needs to be able to make good predictions for (generalize to) examples it has never seen before.

There are two main approaches to generalization: instance-based learning and model-based learning.

Instance-based learning гэж юу вэ
Instance-based learning (IBL) гэдэг нь машин суралцахад хэрэглэгддэг арга бөгөөд шинээр ирсэн өгөгдлийг шууд жишээн дээр тулгуурлан дүгнэх зарчмыг ашигладаг. Өөрөөр хэлбэл, систем нь өмнө нь харсан өгөгдлийн жишээнүүдийг санах, дараа нь шинэ өгөгдөл ирэхэд хамгийн төстэй жишээнүүд дээр үндэслэн шийдвэр гаргах хэлбэрээр ажиллана.


Гол онцлог

Модел үүсгэхэд бага ажил шаардана – Том хэмжээний боловсруулсан дүрэм боловсруулахаас илүү, өгөгдлийг шууд хадгалах.
Шинэ өгөгдөлд хурдан дасан зохицох чадвартай.
Дээрээс доош дүрэм гаргахгүй – Шууд жишээн дээр тулгуурлаж шийддэг.

Жишээ

k-Nearest Neighbors (k-NN) алгоритм – хамгийн түгээмэл instance-based learning жишээ.
Шинэ өгөгдөл ирэхэд k ширхэг хамгийн ойрын жишээг хайж, тэдгээрийн ангилалын ихэнх нь юу байгааг шийдвэрлэнэ.
Locally Weighted Regression – ойрын жишээнүүдийг ашиглан хамаарлыг тодорхойлдог.


Давуу тал:

Энгийн, ойлгоход хялбар.
Том хэмжээний дүрэм боловсруулахаас илүү хурдан хэрэглэх боломжтой.
Өмнөх жишээний мэдээллийг бүрэн ашиглана.

Сул тал:

Их хэмжээний өгөгдөлтэй бол хадгалах, хайлт хийхэд их нөөц шаарддаг.
Өгөгдлийн чанар муутай бол үр дүн буруу гарах магадлал өндөр.


model-based learning and a typical machine learning гэж юу вэ

Model-based learning буюу Модел-д суурилсан суралцах гэдэг нь машин суралцахад өгөгдлөөс дүрэм, загвар (model) боловсруулж, дараагийн шийдвэрийг гаргах аргыг хэлнэ. Өөрөөр хэлбэл, систем нь өмнөх жишээнээс ерөнхий дүрэм боловсруулж, шинэ өгөгдөл ирэхэд тухайн дүрмийн дагуу хариу гаргана

Гол онцлог

Дүрэм буюу загвар боловсруулна – бүх жишээг хадгалахгүй, харин өгөгдлийн бүтэц, хамаарлыг дүрмээр илэрхийлнэ.
Шинэ өгөгдөлд ерөнхийлөлт хийж чадна – шинэ жишээ өмнөх загварт нийцэж байвал таамаглал гаргана.
Хэрэглэхэд илүү хурдан – учир нь загвар нь багтаамж багатай, шууд тооцоолно.

Жишээ

Linear Regression (Шугаман регресс) – өгөгдлийн хамаарлыг шугаман функцээр загварчилна.
Decision Tree (Шийдвэрийн мод) – өгөгдлийг дүрмийн модонд хувиргаж ангилна.
Neural Networks (Мэдрэлийн сүлжээ) – өгөгдлийн нарийн хамаарлыг суралцан ерөнхийлөлт гаргана.

Давуу тал:

Өгөгдлийг хадгалах шаардлага бага.
Шинэ өгөгдөлд сайн ерөнхийлөлт хийж чадна.
Том хэмжээний мэдээллийг боловсруулхад илүү тохиромжтой.

Сул тал:

Загварын оновчтой байдал нь зөв загвар сонгоход шалтгаална.
Хэрэв өгөгдлийн бүтэц төвөгтэй бол загвар буруу байж болно.


Instance-based learning → “Өмнөх жишээ бүрийг санаж, ойрын жишээнээс шийдвэр гаргана.”
Model-based learning → “Өмнөх жишээнээс дүрэм боловсруулаад, дүрмээр шийдвэр гаргана.”
